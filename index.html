01

Платформа, режим и версия синтаксиса

Android — маршрутизация DNS по package_name, исключение приложений из TUN через exclude_package.
Windows / Linux — маршрутизация DNS по process_name. Имя TUN-интерфейса на Windows назначается автоматически.
Фиксированный — один Redirect и один Direct DNS. Минимальный конфиг, предсказуемое поведение.
Fallback — все DNS категории попадают в конфиг. Sing-box перебирает серверы при таймауте. Порядок в списке = приоритет.
Modern — загрузка rule-set через http_client: {"detour": "direct"}. Без warning'ов на 1.14+.
Legacy — загрузка rule-set через download_detour: "direct". Для устройств на 1.11–1.13 (в 1.16 legacy вырежут).
Поле independent_cache не генерируется ни в одной ветке — оно deprecated с 1.14, а его отсутствие валидно и на старых версиях.
02

Системные параметры

Отключены — "log": {"disabled": true}, полное отключение.
panic — только критические сбои. error — для диагностики. debug/trace — только для отладки.
Как часто sing-box скачивает обновлённый .srs-файл с доменами.
Плоский UDP-DNS (порт 53), который резолвит hostname ваших DoH/DoT-серверов в IP-адреса. Без него sing-box не знает, куда подключаться. Проверяйте через dig @IP xbox-dns.ru.
IPv4 only — только A-записи. Убирает таймауты, если провайдер не маршрутизирует IPv6.
Оставьте поле пустым — будет использована ссылка по умолчанию (Dns-rules). Либо укажите свой .srs-файл.
03

Выбор DNS-серверов

Обслуживает приложения из списка, свои домены и домены из Rule-Set. Формат: hostname (DoT/DoH) или полный URL (DoH с кастомным путём).
Обслуживает весь остальной трафик (final). Обычно — быстрый DNS с фильтрацией рекламы.

05

Исключения

Домены, исключённые из Redirect. Пойдут через Direct DNS, даже если приложение или Rule-Set указывает на Redirect. Приоритет выше, чем у правил по приложениям.
Приложения, полностью исключённые из маршрутизации через TUN-интерфейс (inbounds.exclude_package). Их трафик идёт напрямую, минуя sing-box. Только для Android.
