Sing-Box DNS Configurator v3.4

Режим профиля

Фиксированный — один Redirect и один Direct DNS. Минимальный конфиг, предсказуемое поведение.
Fallback — все DNS категории попадают в конфиг. Sing-box перебирает серверы при таймауте. Порядок в списке = приоритет.

Системные параметры

Отключены — "log": {"disabled": true}. Полное отключение, нулевой overhead.
panic — только критические сбои. Рекомендуется для продакшена.
error — ошибки DNS и соединений. Для диагностики.
debug/trace — полный поток. Только для отладки, тормозит.
Как часто sing-box скачивает обновлённый .srs-файл с доменами. Чаще = актуальнее, но больше трафика.
Плоский UDP-DNS (порт 53), который резолвит hostname ваших DoH/DoT-серверов в IP-адреса. Без него sing-box не знает, куда подключаться. Должен быть доступен в вашей сети. Проверяйте через dig @IP xbox-dns.ru.
IPv4 only — возвращает только A-записи. Убирает таймауты на IPv6, если провайдер не маршрутизирует IPv6.
Prefer IPv4 — предпочитает A, но вернёт AAAA если A нет. Может вызывать задержки на IPv6-only сетях.
URL бинарного .srs-файла со списком доменов. Домены из этого файла маршрутизируются в Redirect DNS. Загружается через download_detour: "direct" (без петли).

Выбор DNS-серверов

Обслуживает: приложения из списка, свои домены, домены из Rule-Set. Обычно — SmartDNS или DNS с обходом блокировок. Формат: hostname (DoT/DoH) или полный URL (DoH с кастомным путём).
Обслуживает весь остальной трафик (final). Обычно — быстрый DNS с фильтрацией рекламы (DNSForge, Cloudflare, Quad9).

Маршрутизация в Redirect DNS

Домены, которые пойдут через Redirect DNS. По одному на строку или через запятую. Пример: example.com, sub.domain.org. Пусто = не добавлять.
Android-приложения, чьи DNS-запросы пойдут через Redirect DNS. По одному на строку. Базовые 12 (AI) уже включены. Пусто = только базовые.

Исключения

Домены, исключённые из Redirect. Пойдут через Direct DNS, даже если приложение или Rule-Set указывает на Redirect. Приоритет выше, чем у package_name и rule_set. Пусто = не исключать.
Приложения, полностью исключённые из VPN-туннеля (inbounds.exclude_package). Их трафик идёт напрямую, минуя sing-box. Используйте для WhatsApp, банковских приложений. Пусто = не исключать.
